# AAISM Retake 4-Week Remediation Plan

> Primary objective: raise `AI Technologies and Controls` quickly while maintaining domain balance.

---

## Study Allocation (Weeks 2-4 Priority Model)

- 50%: AI Technologies and Controls
- 30%: AI Governance and Program Management
- 20%: AI Risk Management

Use the split below for a 10-hour week:
- 5.0h Technologies and Controls
- 3.0h Governance and Program Management
- 2.0h Risk Management

---

## Daily Drill Structure (60-120 minutes)

| Block | Duration | Focus |
|---|---:|---|
| Concept Review | 30-60 min | Weakest concept of the day |
| Targeted Questions | 20-30 min | 10-20 questions on same concept |
| Error Log + Recall | 10-15 min | Root-cause tag and 3 key takeaways |

Required rule: remediation for any missed concept within 24-48 hours.

---

## Weekly Cadence

- Mon/Wed/Fri: deep work on Technologies and Controls.
- Tue/Thu: mixed reinforcement with Governance and Risk.
- Sat: 1 longer review block + summary notes.
- Sun: light review or recovery.

---

## Pre-Filled Weekly Plan (30-Day and 60-Day)

### 30-Day Track (Weeks 1-4)

## Week 1 (Baseline + Priority Mapping)
- [ ] Take 1 full timed baseline mock and log it in `progress/05-retake-baseline-and-gap-report.md`.
- [ ] Fill top 5 weak concepts (all from technologies and controls first).
- [ ] Run 3 focused drills (15-20 questions each) on top 2 weak concepts.
- [ ] Build a remediation list for every miss (24-48h loop).

**Weekly output:** baseline score recorded, top 5 weaknesses defined, first remediation cycle started.

## Week 2 (Core Domain Lift)
- [ ] Run 5 technologies-and-controls sessions (concept + questions + error log).
- [ ] Run 2 mixed timed sets (cross-domain transfer).
- [ ] Review all misses within 24 hours and retest missed concepts once.
- [ ] Record one weekly summary of repeated misses.

**Weekly output:** stronger concept recall in weakest domain and fewer repeated misses.

## Week 3 (Integration + Pacing)
- [ ] Run 4 technologies-focused sessions and 2 mixed timed sets.
- [ ] Add at least 2 scenario-style drills combining governance, risk, and controls.
- [ ] Track average time per question and over-time sessions.
- [ ] Perform 1 checkpoint set with strict pacing target.

**Weekly output:** improved cross-domain reasoning and more stable pacing.

## Week 4 (Readiness Checkpoint)
- [ ] Run 1 longer timed mixed set (near full-exam pacing conditions).
- [ ] Validate trend: technologies-and-controls performance up vs week 1.
- [ ] Identify final high-impact weak topics for final polish.
- [ ] Decide: attempt retake soon or extend to 60-day track.

**Weekly output:** go/no-go recommendation for immediate booking vs extension.

### 60-Day Track Extension (Weeks 5-8)

## Week 5 (Advanced Reinforcement)
- [ ] Run 3 technologies sessions, 2 governance sessions, 1 risk reinforcement block.
- [ ] Run 3 mixed timed sets with strict timer.
- [ ] Focus on trap-answer and misread reduction patterns.

**Weekly output:** lower avoidable errors and stronger score consistency.

## Week 6 (Scenario Mastery)
- [ ] Complete 3 scenario-heavy drills (multi-step reasoning).
- [ ] Complete 2 mixed timed sets with post-mortem notes.
- [ ] Rebuild flash notes for top 10 recurring misses.

**Weekly output:** better decision quality on integrated questions.

## Week 7 (Final Readiness Phase I)
- [ ] Run full timed mock #1 and complete detailed post-mortem.
- [ ] Correct all misses with 24-48h remediation.
- [ ] Run 2 short targeted drills on weak clusters found in mock #1.

**Weekly output:** first exam-readiness benchmark captured.

## Week 8 (Final Readiness Phase II)
- [ ] Run full timed mock #2 under strict exam conditions.
- [ ] Compare against mock #1 for trend and stability.
- [ ] Complete go/no-go decision in `progress/07-retake-readiness-gate.md`.

**Weekly output:** booking decision supported by two final mock results.
